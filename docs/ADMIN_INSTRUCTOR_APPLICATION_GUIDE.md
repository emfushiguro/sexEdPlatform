# Admin Guide: Instructor Application Review

## Overview
This guide explains how admins review learner applications to become instructors.

## Accessing Applications
1. Sign in as an admin.
2. Open Instructor Applications from the admin dashboard card or navigate to /admin/instructor-applications.
3. Use status tabs to filter Pending, Approved, and Rejected applications.

## What to Review
Review each application carefully before deciding.

### Tier 1 (All Required)
- Government-issued ID
- NBI or police clearance
- Professional background or bio

### Tier 2 (At Least One Required)
- Teaching credential
- Sex education training certificate
- Professional license

## Approval Checklist
- Applicant identity is clear and documents are readable.
- Clearance is present and acceptable.
- Bio demonstrates relevant educational or professional background.
- At least one Tier 2 credential is valid and supports teaching readiness.

## Rejecting Applications
When rejecting, provide specific and constructive feedback.

### Good rejection reasons
- Document quality is too low to verify identity or credentials.
- Required clearance is missing, expired, or unclear.
- Submitted credentials do not sufficiently demonstrate readiness to teach.
- Application content is incomplete or inconsistent.

### Rejection message quality tips
- Be direct and respectful.
- Explain exactly what needs to be corrected.
- Encourage reapplication after improvements.

## Approval Effects
When you approve:
- User role changes from learner to instructor.
- Instructor role is assigned in permissions.
- Instructor profile is created.
- Role transition is logged for audit.
- Learner data remains preserved.
- Applicant receives an approval notification.

## Rejection Effects
When you reject:
- Application status changes to rejected.
- Rejection reason is saved.
- Applicant receives a rejection notification.
- User remains a learner and can reapply later.
